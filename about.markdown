## 关于这个博客

这是一个基于 Jekyll 和 GitHub Pages 构建的个人技术博客，用于记录学习心得、分享技术文章、展示项目作品。

## 🛠️ 技术栈

| 分类 | 技术 |
|------|------|
| 静态网站 | Jekyll, GitHub Pages |
| 前端 | HTML, CSS, JavaScript |
| 版本控制 | Git, GitHub |
| 自动化部署 | GitHub Actions |

## 📁 项目结构

```
├── _posts/          # 博客文章
├── _layouts/        # 页面布局模板
├── _includes/       # 可复用组件
├── assets/          # 静态资源
├── _config.yml      # Jekyll 配置
└── .github/         # GitHub 配置
```

## 🎯 建站目的

1. **知识沉淀** - 将学习过程中的知识点系统化整理
2. **经验分享** - 把踩坑经历和解决方案记录下来
3. **技术交流** - 与志同道合的开发者互相学习
4. **简历加成** - 展示技术实力和学习能力

## 🌟 特色功能

- **响应式设计** - 适配各种设备和屏幕尺寸
- **深色主题** - 舒适的阅读体验
- **自定义光标** - 炫酷的鼠标跟随效果
- **粒子背景** - 动态背景装饰
- **自动部署** - 代码推送后自动构建发布

## 📝 如何使用

### 本地预览

```bash
# 克隆仓库
git clone https://github.com/11huang-crypto/11huang-crypto.github.io.git

# 进入目录
cd 11huang-crypto.github.io

# 启动 Jekyll 开发服务器
jekyll serve

# 浏览器访问 http://localhost:4000
```

### 发布文章

在 `_posts` 目录下创建新的 Markdown 文件，命名格式：

```
YYYY-MM-DD-title.markdown
```
